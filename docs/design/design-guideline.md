# UI デザインガイドライン

## コンセプト

**「都会的ですれ違う、ミニマルな音楽体験」**

- **ライトモード優先**の両テーマ対応
- **スレート/チャコール**を軸としたモノトーン基調
- Nightlyを参考にした**1画面1目的**の情報設計
- **会話型・体験型**の導線設計

## 情報設計原則（Nightly参考）

### 1画面1目的

各画面は1つの主要タスクに集中させる。

- **オンボーディング**: 1ステップ = 1画面
- **すれ違い詳細**: 曲情報 → コメント → アクション と段階的に
- **設定**: カテゴリごとに画面を分離

### 会話型の導線

UIテキストは語りかけ調を基本とする。

| 従来型 | 会話型 |
|---|---|
| プロフィール設定 | あなたのことを教えてください |
| 楽曲を選択 | 今日シェアしたい曲は？ |
| すれ違い履歴 | 最近出会った音楽 |
| 権限が必要です | 近くの人と出会うために |

### 装飾と操作の分離

- **装飾**（雰囲気づくり）は背景・余白・アニメーションで
- **操作対象**はシンプルに、少なく見せる
- 1画面あたりの主要アクションは**2-3個まで**

## アニメーション・トランジション

### 基本方針

- **控えめで上品** - 過度なアニメーションは避ける
- **意味のある動き** - 状態変化や遷移を分かりやすく
- **一貫性** - アプリ全体で同じイージングを使用

### イージング

| 用途 | イージング | 時間 |
|---|---|---|
| 標準 | ease-out | 200-300ms |
| 進入 | ease-out | 250ms |
| 退出 | ease-in | 200ms |
| 強調 | spring | 300-400ms |

### 特徴的なアニメーション

- **すれ違い成立**: 柔らかいパルスエフェクト
- **いいね**: ハートが軽くバウンス
- **画面遷移**: 横スライド（push/pop）

## アクセシビリティ

### コントラスト比

- テキスト: 最低 **4.5:1**（WCAG AA）
- 大きなテキスト: 最低 **3:1**

### タッチターゲット

- 最小サイズ: **44pt × 44pt**
- ボタン間の余白: **8pt以上**

### ダイナミックタイプ対応

- iOS: Dynamic Type をサポート
- Android: sp 単位を使用
