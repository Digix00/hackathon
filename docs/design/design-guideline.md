# UI デザインガイドライン

## コンセプト

**「都会的ですれ違う、ミニマルな音楽体験」**

- **ライトモード優先**の両テーマ対応
- **スレート/チャコール**を軸としたモノトーン基調
- Nightlyを参考にした**1画面1目的**の情報設計
- **会話型・体験型**の導線設計

---

## デザインの世界観とインスピレーション

### 世界観: 「Urban Serendipity」

都会の雑踏の中で、偶然すれ違った誰かと音楽で繋がる——そんな **静かな高揚感** をデザインで表現する。

| キーワード | 表現方法 |
|---|---|
| 都会的 | モノトーン基調、シャープなタイポグラフィ |
| 偶然性 | パルスリング、波紋エフェクト |
| 音楽 | アルバムアートを主役に、ブラー背景で没入 |
| 繋がり | 2つが出会う瞬間のアニメーション |

### インスピレーション

**参考にしているデザイン**:

| ソース | 取り入れる要素 |
|---|---|
| **Nightly (睡眠アプリ)** | 1画面1目的、会話型UI、大きなタイポグラフィ |
| **Spotify** | アルバムアートを活かしたカラー抽出、ブラー背景 |
| **Linear** | モノトーン配色、余白の使い方、ミニマルなアイコン |
| **Arc Browser** | 洗練されたグラデーション、微細なアニメーション |

### 避けるべきパターン（AIスロップ回避）

| 避ける | 代わりに |
|---|---|
| ありきたりなグラデーション（紫→青） | Slateベースのモノトーン + アルバムカラー |
| 過度なドロップシャドウ | 微細なシャドウ + ブラー背景 |
| 丸すぎる角丸（pill形状の多用） | 適度な角丸（12-16pt） |
| 装飾過多 | 余白で「呼吸」させる |
| 汎用的なアイコン | 音楽体験に特化したシンボル |

### 独自性を生む3つの柱

1. **アルバムアートが主役**
   - アプリのビジュアルはユーザーの音楽で決まる
   - 背景、カラー、雰囲気がダイナミックに変化

2. **「出会いの瞬間」の演出**
   - パルスリング → 予兆 → 成立 → 余韻
   - 他のアプリにはないリアルタイム体験

3. **ミニマルだけど温かい**
   - 機能は絞り込み、情報は厳選
   - でも人との繋がりを感じさせる会話型テキスト

### フォントファミリー

**システムフォント（本文・UI）**:

| プラットフォーム | フォント |
|---|---|
| iOS | SF Pro |
| Android | Roboto |

**アクセントフォント（見出し・強調）**:

| 用途 | フォント | 特徴 |
|---|---|---|
| Hero・大見出し | **Noto Sans JP** (Black 900) | 極太で力強い印象 |
| 曲名・アーティスト名 | **Noto Sans JP** (Medium 500) | 可読性と個性のバランス |
| 英数字アクセント | **Inter** (Tight) または **DM Sans** | モダンでシャープ |

> 💡 **ポイント**: 本文はシステムフォントで可読性を確保しつつ、Hero や曲名には **ウェイトのコントラスト** を活用。極太（Black 900）と細字（Light 300）を組み合わせて視覚的な階層を作る。

**ウェイトコントラストの例**:
```
┌─────────────────────────────────┐
│                                 │
│   今日シェアしている曲           │  ← Light 300, 13pt
│                                 │
│   夜に駆ける                     │  ← Black 900, 28pt
│   YOASOBI                       │  ← Medium 500, 17pt
│                                 │
└─────────────────────────────────┘
```

### フォントサイズ

| 用途 | サイズ | ウェイト | 行間 |
|---|---|---|---|
| 大見出し（Hero） | 32pt | Bold | 1.2 |
| 見出し1 | 24pt | Bold | 1.3 |
| 見出し2 | 20pt | Semibold | 1.3 |
| 見出し3 | 17pt | Semibold | 1.4 |
| 本文（大） | 17pt | Regular | 1.5 |
| 本文 | 15pt | Regular | 1.5 |
| キャプション | 13pt | Regular | 1.4 |
| 補足 | 11pt | Regular | 1.4 |

### 文字サイズの指針（Nightly参考）

- **主要情報は大きく明快に** - 睡眠前でも読めるNightlyの設計を参考に、すれ違い通知や曲名は大きく表示
- **階層は3段階まで** - プライマリ、セカンダリ、ターシャリの3段階で情報を整理

## アルバムアートワーク

ジャケット画像はアプリの視覚的魅力の中核。以下のパターンを組み合わせて活用する。

### 1. 背景ブラー効果

ジャケットを拡大・ぼかして背景に使用し、没入感を演出。

```
実装:
- ぼかし半径: 40-60pt
- 拡大率: 150-200%（画面を覆う）
- 暗めオーバーレイ: rgba(0,0,0,0.3)〜0.5
- 彩度: 80-90%（やや落とす）
```

**使用箇所**:
- すれ違い詳細画面（ヒーロー領域）
- 曲選択モーダル
- プレビュー再生時

### 2. ドミナントカラー抽出

ジャケットから主要色を抽出し、UIアクセントに動的適用。

```
抽出する色:
- Primary: 最も面積の大きい色
- Secondary: 2番目に多い色
- Accent: 最もコントラストのある色
```

**適用パターン**:
| 要素 | 適用方法 |
|---|---|
| カードボーダー | Primary を 20% 透明度で |
| グロー効果 | Primary でソフトシャドウ |
| ボタン背景 | Accent（テキスト読みやすさ確保） |
| グラデーション | Primary → 透明 |

**フォールバック**:
- 抽出失敗時・読み込み中はデフォルトの Slate を使用

### 3. ヒーロー表示

ホーム画面でシェア中の曲を大きく見せる。

```
┌─────────────────────────────┐
│                             │
│   ┌───────────────────┐     │
│   │                   │     │
│   │   アルバムアート   │     │  ← 120pt × 120pt
│   │                   │     │
│   └───────────────────┘     │
│                             │
│   今日シェアしている曲       │
│   曲名                      │
│   アーティスト              │
└─────────────────────────────┘

サイズ: 120pt × 120pt（ホーム）/ 200pt（詳細）
角丸: 12pt
シャドウ: 0 8pt 24pt rgba(0,0,0,0.15)
```

### 4. コラージュ/グリッド表示

複数のジャケットを並べて「出会った音楽の軌跡」を可視化。

```
┌──┐┌──┐┌──┐┌──┐
│  ││  ││  ││  │  ← 各 56pt × 56pt
└──┘└──┘└──┘└──┘
┌──┐┌──┐┌──┐
│  ││  │+ N│
└──┘└──┘└──┘

グリッド仕様:
- アイテムサイズ: 56pt × 56pt
- 角丸: 6pt
- ギャップ: 8pt
- 最大表示: 7枚 + 「+N」バッジ
```

**インタラクション**:
- タップで該当のすれ違い詳細へ
- 長押しでクイックプレビュー（曲名ポップアップ）

### 5. 通知トースト

すれ違い発生時、ジャケット付きで通知。

```
┌─────────────────────────────────┐
│ ┌────┐                         │
│ │ 🎵 │  新しいすれ違い！        │
│ │    │  曲名 - アーティスト     │
│ └────┘                         │
└─────────────────────────────────┘

ジャケットサイズ: 40pt × 40pt
角丸: 6pt
背景: blur + カード背景色
```

### パフォーマンス考慮

| 処理 | 方針 |
|---|---|
| サムネイル | 3段階用意（56pt / 120pt / 200pt） |
| ブラー背景 | サーバーサイド事前生成 or クライアントキャッシュ |
| カラー抽出 | APIレスポンスに含める（dominant_colors フィールド） |
| プレースホルダー | Slate 200 矩形 + シマーアニメーション |

## 情報設計原則（Nightly参考）

### 1画面1目的

各画面は1つの主要タスクに集中させる。

- **オンボーディング**: 1ステップ = 1画面
- **すれ違い詳細**: 曲情報 → コメント → アクション と段階的に
- **設定**: カテゴリごとに画面を分離

### 会話型の導線

UIテキストは語りかけ調を基本とする。

| 従来型 | 会話型 |
|---|---|
| プロフィール設定 | あなたのことを教えてください |
| 楽曲を選択 | 今日シェアしたい曲は？ |
| すれ違い履歴 | 最近出会った音楽 |
| 権限が必要です | 近くの人と出会うために |

### 装飾と操作の分離

- **装飾**（雰囲気づくり）は背景・余白・アニメーションで
- **操作対象**はシンプルに、少なく見せる
- 1画面あたりの主要アクションは**2-3個まで**

## アニメーション・トランジション

### 基本方針

- **控えめで上品** - 過度なアニメーションは避ける
- **意味のある動き** - 状態変化や遷移を分かりやすく
- **一貫性** - アプリ全体で同じイージングを使用

### イージング

| 用途 | イージング | 時間 |
|---|---|---|
| 標準 | ease-out | 200-300ms |
| 進入 | ease-out | 250ms |
| 退出 | ease-in | 200ms |
| 強調 | spring | 300-400ms |

### 特徴的なアニメーション

- **すれ違い成立**: 柔らかいパルスエフェクト
- **いいね**: ハートが軽くバウンス
- **画面遷移**: 横スライド（push/pop）

詳細なマイクロインタラクション、ジェスチャー、トランジションのタイミングは [インタラクション定義](./interactions.md) を参照。

## このアプリ独自の体験

### すれ違いリアルタイム演出

このアプリの核となる体験。すれ違いの「瞬間」を特別なものとして演出する。

**体験フロー**:
```
[待機状態] → [予兆] → [すれ違い成立] → [余韻] → [詳細へ]
```

| フェーズ | 演出 |
|---|---|
| 待機 | パルスリング（ゆっくり脈打つ円） |
| 予兆 | パルス加速、波打ち、外周に小さな点 |
| 成立 | 光の粒子放射、3重波紋、カード出現 |
| 余韻 | 残光エフェクト、特別なカード表示 |

**ハプティクス**: すれ違い成立時に触覚フィードバック

詳細は [すれ違いリアルタイム演出](./screens/encounter-realtime.md) を参照。

## 背景デザイン

単色の背景ではなく、**レイヤーを重ねて奥行きを出す**ことで洗練された印象を与える。

### グラデーション背景

| 用途 | 種類 | 仕様 |
|---|---|---|
| ホーム画面 | Radial | 中央から外側へ Slate 50 → Slate 100（ライト）|
| すれ違い成立時 | Linear | 上から下へ Primary色(20%) → 透明 |
| カード背景 | Subtle | White → Slate 50（5%程度の変化）|

**ライトモード例**:
```css
/* ホーム画面 */
background: radial-gradient(
  circle at 50% 30%,
  #FFFFFF 0%,
  #F8FAFC 50%,
  #F1F5F9 100%
);

/* すれ違い詳細（アルバムカラー適用時） */
background: linear-gradient(
  180deg,
  rgba(var(--dominant-color), 0.15) 0%,
  transparent 40%
);
```

### 幾何学的パターン

控えめな幾何学模様で空間に奥行きを追加。

| パターン | 用途 | 仕様 |
|---|---|---|
| ドットグリッド | 空状態の背景 | 8pt間隔、Slate 200、opacity 30% |
| 同心円 | すれ違い待機 | パルスリングと連動、Slate 100 |
| ノイズテクスチャ | カード背景 | 微細なノイズ、opacity 3-5% |

**ドットグリッドの実装例**:
```css
background-image: radial-gradient(
  circle,
  #CBD5E1 1px,
  transparent 1px
);
background-size: 24px 24px;
opacity: 0.3;
```

### レイヤー構成

画面の奥行きを3層で構成:

```
┌───────────────────────────────────┐
│ Layer 3: コンテンツ（カード、テキスト） │  ← 最前面
├───────────────────────────────────┤
│ Layer 2: パターン/テクスチャ         │  ← 微細な模様
├───────────────────────────────────┤
│ Layer 1: グラデーション背景          │  ← 最背面
└───────────────────────────────────┘
```

### すれ違い詳細のブラー背景（強化版）

アルバムアートを使った没入感のある背景:

```
背景構成:
1. アルバムアート（blur: 60pt, scale: 200%）
2. グラデーションオーバーレイ（上: 透明 → 下: 背景色 60%）
3. ノイズテクスチャ（opacity: 3%）
4. ビネット効果（四隅を暗く）
```
