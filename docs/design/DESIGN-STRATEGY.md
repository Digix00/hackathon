# デザイン戦略: 完成度を最大化する

## 前提: スケジュール制約

```
Day 4 (3/18): ホーム画面 + 履歴画面 + 通知
Day 5 (3/19): UI調整・アニメーション (13:00-16:00 = 3時間のみ)
Day 6 (3/20): 発表資料作成
```

**UI に使える時間は実質 1.5日**。後から直す時間はない。

---

## 戦略: 「最初から完成形で作る」

### 原則

| NG | OK |
|---|---|
| とりあえず動かす → 後で綺麗に | 最初から完成形のUIで作る |
| 全画面を均等に | デモで見せる画面だけ完璧に |
| 機能を増やす | 見た目を磨く |

---

## デモシナリオと優先度

```
1. アプリ起動 → オンボーディング（権限許可）     [優先度: 中]
2. Sign in with Apple でログイン               [優先度: 低]
3. お気に入り曲を設定（Spotify 検索 → 選択）    [優先度: 中]
4. 2台を近づける → BLE ですれ違い発生          [優先度: -]
5. プッシュ通知が届く                          [優先度: 低]
6. すれ違い履歴を開く → 相手の曲が表示される    [優先度: 高]
7. 曲詳細を見る                                [優先度: 最高]
```

### 完成度を集中させる画面

| 優先度 | 画面 | 理由 |
|---|---|---|
| ★★★ | **すれ違い詳細** | デモのクライマックス。一番長く映る |
| ★★☆ | **ホーム画面** | デモの起点。第一印象 |
| ★☆☆ | 履歴一覧 | リストだけなので簡素でOK |
| ☆☆☆ | オンボーディング | 流すだけ。機能すればOK |
| ☆☆☆ | 設定 | デモで見せない |

## Day 4: 「最初から完成形」で作る

### ホーム画面（曽根 13:00-16:00）

**最初からこの品質で作る**:

```
余白: たっぷり取る（上下左右 24pt）
カード: 角丸 16pt、影 (0 4pt 16pt rgba(0,0,0,0.08))
フォント:
  - 見出し: 20pt Semibold
  - 本文: 15pt Regular
  - 補足: 13pt Regular, Slate 500
```

**シェア中の曲カード**:
```swift
// 最初からこのスタイルで実装
VStack {
    albumArt
        .frame(width: 64, height: 64)
        .cornerRadius(8)

    Text(trackName)
        .font(.system(size: 17, weight: .semibold))

    Text(artistName)
        .font(.system(size: 15))
        .foregroundColor(.secondary)
}
.padding(20)
.background(Color.white)
.cornerRadius(16)
.shadow(color: .black.opacity(0.08), radius: 16, y: 4)
```

## Day 5: 「すれ違い詳細」だけ磨く

### UI調整・アニメーション（曽根 13:00-16:00 = 3時間）

この3時間で以下だけを完璧にする:

### 1. すれ違い詳細画面のレイアウト（1時間）

```
┌─────────────────────────────┐
│ ←                          │
│                             │
│      ┌───────────────┐      │
│      │               │      │
│      │  アルバム      │      │  ← 160pt × 160pt（大きく）
│      │  アート        │      │
│      │               │      │
│      └───────────────┘      │
│                             │
│         曲名                │  ← 24pt Bold
│      アーティスト            │  ← 17pt, Slate 500
│                             │
│   [▶ 再生]     [♡ いいね]   │  ← 横並び、高さ 48pt
│                             │
│   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                             │
│   👤 ユーザー名さん          │
│      今日 10:30 にすれ違い   │
│                             │
└─────────────────────────────┘

ポイント:
- アルバムアートを大きく（プロのデザイン素材を活かす）
- 余白をたっぷり
- 情報は3階層まで
```

### 2. いいねボタンのアニメーション（30分）

```swift
// タップ時にハートがバウンス
withAnimation(.spring(response: 0.3, dampingFraction: 0.6)) {
    isLiked.toggle()
    scale = 1.3
}
withAnimation(.spring(response: 0.2, dampingFraction: 0.8).delay(0.1)) {
    scale = 1.0
}

// ハプティクス
UIImpactFeedbackGenerator(style: .light).impactOccurred()
```

これだけで「細部まで作り込んでる」感が出る。

### 3. 画面遷移（30分）

```swift
// 履歴 → 詳細 のトランジション
// アルバムアートが拡大するシェアドエレメント風

NavigationLink {
    DetailView()
} label: {
    ListRow()
}
.matchedGeometryEffect(id: track.id, in: namespace)
```

### 4. ブラー背景（30分）

詳細画面の背景にアルバムアートをぼかして敷く:

```swift
ZStack {
    // 背景ブラー
    AsyncImage(url: albumArtURL)
        .aspectRatio(contentMode: .fill)
        .blur(radius: 50)
        .overlay(Color.black.opacity(0.4))
        .ignoresSafeArea()

    // コンテンツ
    VStack {
        // ...
    }
}
```

### 5. 残り時間でサウンド追加（30分）

すれ違い成立時の通知音:
- フリー素材で「ベル音」「ポップ音」を探す
- 0.5秒以内の短い音
- なくてもOK、あれば加点

## デモ用データの準備（河田担当）

デモ映えするデータを用意:

### 曲の選定

| 基準 | 理由 |
|---|---|
| 知名度が高い | 審査員が「あの曲だ」と分かる |
| ジャケットが綺麗 | 見た目の完成度に直結 |
| 日本語 | 親しみやすい |

**候補**:
- YOASOBI「夜に駆ける」
- 米津玄師「Lemon」
- Official髭男dism「Pretender」
- あいみょん「マリーゴールド」

### ユーザー名

自然な名前:
- 「ゆうき」「あおい」「はると」
- 「music_lover」のような英語も可

---

## やらないこと（明示的に除外）

| 項目 | 理由 |
|---|---|
| ダークモード | 工数がかかる、見た目だけ |
| パルスリング | 複雑、なくても体験は伝わる |
| コメント機能 | スコープ外 |
| 通知センター | プッシュだけで十分 |
| 設定画面の細部 | デモで見せない |
| 空状態の詳細UI | シンプルなメッセージで十分 |
