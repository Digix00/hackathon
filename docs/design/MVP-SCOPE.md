# MVP UI スコープ（1週間ハッカソン用）

## 方針

**「動くコア体験」を優先。細部は後回し。**

- ライトモードのみ（ダークモード対応なし）
- アニメーションは最小限
- エッジケースは簡易対応
- 発表で見せられるメインフローを完成させる

---

## 実装する / しない

### 実装する（Must）

| 画面 | 内容 | 工数目安 |
|---|---|---|
| スプラッシュ | ロゴ表示のみ（アニメなし） | 0.5日 |
| オンボーディング | 認証 → プロフィール → 権限 → 完了（4画面） | 1日 |
| ホーム | すれ違い数 + 最近のリスト + シェア曲表示 | 1日 |
| すれ違い詳細 | 曲情報 + いいねボタン + ユーザー情報 | 1日 |
| すれ違い履歴 | シンプルなリスト表示 | 0.5日 |
| 設定 | プロフィール編集 + ログアウトのみ | 0.5日 |
| 検索 | 曲検索（結果表示 + 選択） | 0.5日 |
| 通知 | すれ違い成立のプッシュ通知 | 0.5日 |

**合計: 約5.5日**（バッファ1.5日）

### 実装しない（後回し）

| 項目 | 理由 |
|---|---|
| ダークモード | 工数がかかる、見た目だけ |
| パルスリング・波紋アニメ | 複雑、なくても体験は伝わる |
| 予兆表示 | コア体験ではない |
| コメント機能 | あれば良いが必須ではない |
| ブロック/ミュート/通報 | 審査用、ハッカソンでは不要 |
| 空状態の詳細提案 | シンプルなメッセージで十分 |
| 通知センター | プッシュ通知だけで十分 |
| 通知設定 | デフォルト固定でOK |
| 他ユーザープロフィール | タップしても何も起きなくてOK |
| プログレスバー（オンボ） | なくても進行は分かる |

## 簡略化した画面仕様

## 発表で見せるフロー

1. **オンボーディング** → 認証 → 設定完了
2. **ホーム画面** → シェア曲が表示されている
3. **すれ違いデモ** → 2台の端末で実演 → 通知が届く
4. **詳細画面** → 曲を確認 → いいね → 再生

このフローが動けば、ハッカソンとしては十分。
