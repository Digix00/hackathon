# スプラッシュ・検索・プロフィール UI設計

## 概要

MVP必須のUI要素を定義する。

---

# 1. スプラッシュ画面

## 目的

アプリ起動時の第一印象。ブランドを伝えつつ、ローディングを待たせる。

## レイアウト

```
┌─────────────────────────────┐
│                             │
│                             │
│                             │
│                             │
│         [アプリロゴ]         │
│                             │
│                             │
│                             │
│                             │
│                             │
│                             │
│                             │
└─────────────────────────────┘
```

## 仕様

| 要素 | 仕様 |
|---|---|
| 背景 | Slate 50（ライト）/ Slate 900（ダーク） |
| ロゴサイズ | 80pt × 80pt |
| ロゴ位置 | 中央、やや上寄り（画面高さの40%地点） |
| 表示時間 | 最低 800ms、ローディング完了まで |

## アニメーション

```
0ms        400ms       800ms+
│           │           │
▽           ▽           ▽
opacity 0   opacity 1   フェードアウト
scale 0.9   scale 1.0   → ホーム画面へ
```

**ロゴ出現**:
- opacity: 0 → 1（300ms ease-out）
- scale: 0.95 → 1.0（300ms ease-out）

**画面遷移**:
- スプラッシュ全体がフェードアウト（250ms）
- ホーム画面がフェードイン（250ms）

## ダークモード

- システム設定に追従
- 起動時点のモードで表示

## ロゴ仮仕様

```
┌───────────────┐
│               │
│    ∿  🎵     │   音符 + 波紋のモチーフ
│               │
└───────────────┘

カラー: Slate 900（ライト）/ Slate 50（ダーク）
```

## 2.5 インタラクション

| 操作 | 挙動 |
|---|---|
| 検索フィールドタップ | キーボード出現、フォーカス |
| 文字入力 | 300ms デバウンス後に検索実行 |
| 結果タップ | 曲を選択リストに追加 |
| ✕タップ | 選択から削除 |
| 決定 | 前画面に戻り、選択を反映 |
| キャンセル | 変更を破棄して戻る |

## 3.3 表示情報

| 情報 | 表示条件 | 表示形式 |
|---|---|---|
| アバター | 常時 | 80pt 円形 |
| ユーザー名 | 常時 | 20pt Semibold |
| ひとこと | 設定時のみ | 15pt Regular |
| 出身地 | 公開設定時 | 📍 + テキスト |
| 年代 | 公開設定時 | 🎂 + 「20代」等 |
| すれ違い回数 | 常時 | 🎵 + 「すれ違い N回」 |
| シェア中の曲 | 常時 | 曲カード |

## 3.4 安全機能（ワンタップ）

```
┌───────────┐ ┌─────────┐
│  ブロック  │ │  通報   │
└───────────┘ └─────────┘
```

**ブロック**:
- タップ → 確認ダイアログ（シンプル）
```
ブロックしますか？
このユーザーとすれ違わなくなります。

[キャンセル] [ブロック]
```
- 確認後即実行、トースト「ブロックしました」

**通報**:
- タップ → 理由選択シート
```
通報理由を選択
┌─────────────────────────┐
│ スパム・迷惑行為        │
├─────────────────────────┤
│ 不適切なコンテンツ      │
├─────────────────────────┤
│ なりすまし             │
├─────────────────────────┤
│ その他                 │
└─────────────────────────┘
        キャンセル
```
- 選択後即送信、トースト「通報しました」

## 3.5 スタイル

```
モーダル:
  高さ: 画面の70%
  角丸: 上部 20pt
  背景: カード背景色
  ドラッグインジケーター: 36pt × 4pt、Slate 300

  アイコン: 16pt、Slate 400
```

# 4. ミュート機能の追加

ブロックほど強くない「非表示」機能。

```
┌───────────┐ ┌───────────┐ ┌─────────┐
│  ミュート  │ │  ブロック  │ │  通報   │
└───────────┘ └───────────┘ └─────────┘
```

**ミュート**:
- すれ違い通知を受け取らない
- 履歴には表示される（薄くグレーアウト）
- 確認なしで即実行、トースト「ミュートしました」

**ブロック**:
- すれ違い自体が成立しない
- 履歴からも非表示
- 確認ダイアログあり
