# オンボーディング画面 UI設計

## 概要

初回起動時のユーザー体験を設計。**3ステップ以内**で初期設定を完了させ、すぐにアプリ体験を開始できるようにする。

---

## フロー構成（最適化版）

```
[ウェルカム + 認証] → [プロフィール + 音楽] → [権限（統合）] → [完了]
```

**計4画面**。進行度インジケーターで常に「あと何ステップか」を明示。

### 設計原則

| 原則 | 適用 |
|---|---|
| 3ステップ以内 | 認証後の設定を3画面に集約 |
| 進行度明示 | 全画面にプログレスバー表示 |
| 1画面1目的 | 各画面で求めるアクションは1つ |
| 会話型 | 指示ではなく共感の文言 |

---

- 3つの権限を1画面でカード形式に

- 拒否しても進行可能
- 制限される体験を具体的に説明
- 「設定で許可」で後から変更可能なことを明示

## プログレスインジケーター

### デザイン

```
認証後に表示（3ステップ）:

ステップ1: ━━━━━━━━○━━━━━━━━━━━━━━━
ステップ2: ━━━━━━━━━━━━━━○━━━━━━━━━
ステップ3: ━━━━━━━━━━━━━━━━━━━━━━●

高さ: 4pt
アクティブ色: Slate 900（ライト）/ Slate 100（ダーク）
非アクティブ色: Slate 200（ライト）/ Slate 700（ダーク）
位置: Safe Area 下、16pt マージン
```

### アニメーション

```
次の画面へ:
- プログレス伸長: 300ms ease-out
- 完了時: ●マーカーがパルス（1回）
```

## 権限拒否時の代替体験詳細

| 権限 | 拒否時の体験 | 後からの誘導 |
|---|---|---|
| 位置情報 | BLEのみで検知（精度低下） | ホームに「精度を上げる」バナー |
| Bluetooth | すれ違い不可 | ホームに「すれ違いを有効にする」カード |
| 通知 | すれ違い時に通知なし | 設定画面で「通知をオンに」促進 |

### Bluetooth拒否時（必須機能）

```
┌─────────────────────────────┐
│                             │

│     Bluetoothが必要です     │

│                             │

│   送信されることはありません  │
│                             │
│    ┌─────────────────┐      │
```

## 状態管理

### 中断・再開

```
保存タイミング:
- 認証完了時
- 各設定値変更時

再開時:
- 最後に完了したステップの次から開始
- 入力済み値は保持
```

### スキップ可能な項目

| 項目 | スキップ可否 | スキップ時の挙動 |
|---|---|---|
| 写真 | ○ | デフォルトアバター |
| 曲選択 | ○ | ホームで設定促進 |
| 権限（個別） | ○ | 代替体験を提供 |
| ニックネーム | × | 必須入力 |

---

## エラー処理

| シナリオ | 対応 |
|---|---|
| 認証失敗 | ボタン下にインラインエラー「接続できませんでした」 |
| ネットワークエラー | 画面下部にバナー + リトライボタン |
| ニックネーム使用済み | フィールド下にエラー + 代替案提示 |

## アクセシビリティ

### VoiceOver/TalkBack

| 要素 | 読み上げ |
|---|---|
| プログレス | 「ステップ1/3」 |
| 権限カード | 「位置情報、許可が必要です。近くにいる人を見つけるために使います」 |
| 許可済み | 「位置情報、許可済み」 |

### ダイナミックタイプ

- 大きいサイズでもプログレスバーは固定高さ
- テキストは最大150%まで対応
