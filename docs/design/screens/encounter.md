# すれ違い履歴・詳細画面 UI設計

## 概要

すれ違いの履歴一覧と、個別の詳細画面を設計。曲情報の閲覧、いいね、コメント、通報などのインタラクションを提供する。

---

## 画面一覧

1. **履歴一覧画面** - すれ違いの時系列リスト
2. **詳細画面** - 個別のすれ違い詳細とコメント

---

## セクション詳細

### ナビゲーションバー
- 左: 戻るボタン（ホームから遷移時）またはタイトルのみ（タブから）
- 中央: 「すれ違い履歴」
- 右: 検索アイコン

### 日付セクションヘッダー

**表示ルール**:
| 日付 | 表示 |
|---|---|
| 当日 | 今日 |
| 前日 | 昨日 |
| 今週内 | 曜日（月曜日など） |
| それ以前 | 日付（3月12日） |

**スタイル**:
```
フォント: 13pt Semibold
色: Slate 500
マージン: 上24pt、下8pt
背景: sticky（スクロール時固定）
```

### リストアイテム

```
┌─────────────────────────────────────────┐
│  ┌────┐                                │
│  │    │  曲名                 ユーザー名 │
│  │ 🎵 │  アーティスト名                 │
│  │    │                        10:30   │
│  └────┘                                │
└─────────────────────────────────────────┘
```

**スタイル**:
```
高さ: 80pt
パディング: 16pt
アルバムアート: 48pt × 48pt、角丸 6pt
曲名: 15pt Semibold
アーティスト: 13pt Regular、Slate 500
ユーザー名: 13pt Regular、右寄せ
時刻: 13pt Regular、Slate 400、右下
区切り線: 下部 1pt、左インデント 80pt
```

**いいね済み表示**:
- 曲名の右にハートアイコン（小）を表示

### 検索機能

検索アイコンタップで検索バー展開:
```
┌─────────────────────────────┐
│ 🔍 曲名、アーティスト、ユーザー │
└─────────────────────────────┘
```

- リアルタイムフィルタリング
- 検索対象: 曲名、アーティスト名、ユーザー名

## セクション詳細

### ナビゲーションバー（AppScaffold）

**構成**:
```swift
AppScaffold(
  title: encounter.track.title,
  subtitle: "3 MINUTES AGO • SHIBUYA STATION",
  accentColor: encounter.track.color
)
```

**スタイル**:
- タイトル: 曲名
- サブタイトル: 相対時間 + 場所（大文字、Kerning適用）
- 左: 戻るボタン
- 右: メニュー（•••）→ Share/Mute/Block/Report

### Main Track Info Card

**レイアウト**:
```
VStack(spacing: 24):
  1. MockArtworkView（160pt、シャドウ付き）
  2. VStack(曲名 + アーティスト名)
  3. ユーザーカードボタン

パディング: 縦12pt
配置: 中央揃え
```

**アルバムアート**:
```
サイズ: 160pt × 160pt
シャドウ: encounter.track.color 30%透明度、radius 30pt、y 15pt
角丸: MockArtworkView デフォルト
```

**曲名・アーティスト**:
```
曲名:
  フォント: PrototypeTheme.Typography.Encounter.screenTitle
  色: テキストプライマリ
  Tracking: -0.5

アーティスト:
  フォント: PrototypeTheme.Typography.Encounter.sectionTitle
  色: テキストセカンダリ

間隔: 8pt
```

### ユーザーカード（ボタン）

**レイアウト**:
```
Button:
  HStack(spacing: 8):
    MockArtworkView(24pt、person.fill)
    Text(encounter.userName)  // action
    Image(chevron.right)      // metaCompact

背景: Surface 60%透明度、Capsule
パディング: 横16pt、縦10pt
色: テキストプライマリ
```

**タップアクション**:
- `showsProfile = true`
- シート表示: OtherUserProfileView()

### Lyric Fragment（歌詞フラグメント）

**目的**: すれ違いに紐づく歌詞の一部を表示

**レイアウト**:
```
GlassmorphicCard:
  VStack(alignment: .leading, spacing: 16):
    HStack:
      Image(quote.opening)  // encounter.track.color
      Spacer
      Text("LYRIC FRAGMENT")  // eyebrow、Kerning 1.2

    Text(encounter.lyric)  // 22pt Bold Italic
      色: テキストプライマリ
      行間: 4pt

    HStack:
      Spacer
      Image(quote.closing)  // encounter.track.color
```

**GlassmorphicCard スタイル**:
- 半透明マテリアル背景
- 角丸: 16pt
- パディング: 16-20pt

### アクションボタン

**レイアウト**:
```
VStack(spacing: 16):
  HStack(spacing: 12):
    Button(LIKE)           // 左、拡張
    Button(ellipsis)       // 右、52pt固定

  PrimaryButton(ADD YOUR LYRIC)  // 全幅
```

**LIKEボタン**:
```
HStack:
  Image(heart.fill)
  Text("LIKE")

フォント: action
色: 白
背景: encounter.track.color
サイズ: maxWidth × 52pt
角丸: 16pt、continuous
```

**その他ボタン（ellipsis）**:
```
Image(ellipsis)
  フォント: sectionTitle

背景: Surface Muted
サイズ: 52pt × 52pt
角丸: 16pt、continuous
```

**ADD YOUR LYRIC ボタン**:
```
PrimaryButton(

```

## アニメーション

### LIKEボタンタップ（実装予定）
1. ハートアイコンが一瞬拡大（1.2倍）
2. バウンスして元のサイズに
3. 色が変化（グレー → encounter.track.color）

### モーダル表示
- シート: .presentationDetents([.medium, .large])
- スライドアップアニメーション
- ドラッグで閉じる

### 画面遷移
- 履歴一覧→詳細: NavigationStack の標準プッシュ遷移
- 右からスライドイン

---

## テーマとカラー

**アクセントカラー**:
- `encounter.track.color` を使用
- LIKEボタン背景、歌詞のquoteアイコンに適用

**PrototypeTheme を使用**:
```
background: 背景色
surface: カード背景
surfaceElevated: 強調カード
surfaceMuted: ミュート背景

textPrimary, textSecondary, textTertiary
accent, border
success, warning, error, info
```
