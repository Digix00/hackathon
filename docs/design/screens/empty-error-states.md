# 空状態・エラー状態 UI設計

## 概要

すれ違いが少ない時や、エラー発生時でもユーザーを放置せず、**次の行動を提案**する。ネガティブな状態をポジティブな体験に変える。

---

## 設計原則

| 原則 | 適用 |
|---|---|
| 放置しない | 必ず次のアクションを提示 |
| 責めない | ユーザーのせいにしない文言 |
| 具体的に | 「時間帯」「場所」「設定」を具体提案 |
| 共感型 | 指示ではなく寄り添う |

---

# 1. 空状態（Empty States）

## 1.1 すれ違いゼロ（初回）

初めてアプリを使い始めた直後。

```
┌─────────────────────────────┐
│                             │
│                             │
│          🚶               │
│                             │
│    まだすれ違いがありません   │
│                             │
│    ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                             │
│    出会いやすい時間帯         │
│                             │
│    🕐 通勤・通学の時間       │
│       （7-9時、17-20時）    │
│                             │
│    📍 人が多い場所           │
│       駅、カフェ、商業施設    │
│                             │
│                             │
│    ┌─────────────────┐      │
│    │  検知範囲を広げる  │      │
│    └─────────────────┘      │
│                             │
└─────────────────────────────┘
```

**要素**:
- イラスト: 歩く人（期待感のある表現）
- メッセージ: 「まだすれ違いがありません」（責めない）
- おすすめ行動: 時間帯・場所を具体的に提案
- CTA: 設定変更への誘導

# 5. コピーライティング

## 空状態

| 状態 | NG | OK |
|---|---|---|
| すれ違いゼロ | すれ違いがありません | まだすれ違いがありません |
| 検索ゼロ | 結果がありません | 見つかりませんでした |
| いいねゼロ | いいねがありません | まだいいねした曲がありません |

**「まだ」を使う**: これからある可能性を示唆

## エラー

| 状態 | NG | OK |
|---|---|---|
| ネットワーク | エラーが発生しました | 接続できませんでした |
| サーバー | サーバーエラー | 問題が発生しました |
| 権限 | 許可が必要です | 〇〇するには許可が必要です |

**具体的に**: 何が起きたか、何をすればいいかを明確に
