# すれ違いリアルタイム演出 UI設計

## コンセプト

**「音楽が出会う瞬間を、体感する」**

すれ違いは一瞬の出来事。その瞬間を特別な体験として演出し、「誰かの音楽と出会った」という実感を与える。他のアプリにはない、このアプリだけの体験。

---

## 体験フロー

```
[待機状態] → [予兆] → [すれ違い成立] → [余韻] → [詳細へ]
```

---

# 1. 待機状態（Standby）

## ホーム画面の「パルスリング」

アプリがバックグラウンドで検知中であることを、ホーム画面で常に感じられるようにする。

```
┌─────────────────────────────┐
│                             │
│                             │
│         ╭─────╮             │
│        ╱  ◉    ╲            │  ← パルスリング
│       │   あなた  │           │     ゆっくり脈打つ円
│        ╲        ╱            │
│         ╰─────╯             │
│                             │
│     すれ違いを探しています    │
│                             │
└─────────────────────────────┘
```

### パルスアニメーション

```
状態: 検知中（通常）
─────────────────────────
  ◯ → ○ → ◎ → ○ → ◯

周期: 3秒
不透明度: 0.3 → 0.6 → 0.3
スケール: 1.0 → 1.15 → 1.0
```

### デザイン仕様

| 要素 | 仕様 |
|---|---|
| リングサイズ | 120pt |
| リング太さ | 2pt |
| リング色 | Slate 300（ライト）/ Slate 600（ダーク） |
| 中心アイコン | 音符 or ユーザーアバター |
| アニメーション | ease-in-out, 3秒周期 |

### 状態表示テキスト

| 状態 | テキスト |
|---|---|
| 検知中 | すれ違いを探しています |
| Bluetooth オフ | Bluetoothをオンにしてください |
| 位置情報オフ | 位置情報を許可してください |
| 一時停止中 | すれ違い検知を一時停止中 |

# 2. 予兆（Approaching）

## 近くに誰かがいる気配

BLE信号を検知したが、まだすれ違い成立前の状態。

```
┌─────────────────────────────┐
│                             │
│         ╭─────╮             │
│        ╱ ∿∿∿∿  ╲            │  ← 波紋が少し乱れる
│       │   ◉    │            │
│        ╲ ∿∿∿∿  ╱            │
│         ╰─────╯             │
│              ·              │  ← 外周に小さな点
│                             │
│      近くに誰かいるかも...    │
│                             │
└─────────────────────────────┘
```

### アニメーション変化

```
通常 → 予兆
─────────────────────────
パルス周期: 3秒 → 1.5秒（速くなる）
リング: 均一 → 微細な波打ち
外周: なし → 小さな点が1-2個出現
```

### 波打ちアニメーション

```css
/* リングの輪郭が微かに揺れる */
border-radius: 50% → 48% 52% 50% 50% → 50% 48% 52% 50% → 50%
周期: 2秒
```

### 仕様

| 要素 | 仕様 |
|---|---|
| 外周の点 | 4pt、Slate 400、ゆっくり移動 |
| パルス加速 | 線形に 3秒 → 1.5秒 |
| テキスト | フェードで切り替え |
| ハプティクス | なし（まだ確定前なので控えめ） |

# 3. すれ違い成立（Encounter）

## 核心体験：音楽が出会う瞬間

すれ違いが成立した瞬間を**短く気持ちよく**演出する。

### 設計原則

| 原則 | 適用 |
|---|---|
| 短く | 全体で600ms以内に完結 |
| 気持ちよく | パルス + 軽いバウンスで心地よさ |
| 2択に絞る | 「聴く」or「いいね」のみ |

### 演出シーケンス（0-600ms）

```
0ms        200ms       400ms       600ms
│           │           │           │
▽           ▽           ▽           ▽
パルス収縮   バウンス     カード出現   完了
+ ハプティクス  + 波紋1つ
```

### Phase 1: パルス + ハプティクス（0-200ms）

```
┌─────────────────────────────┐
│                             │
│         ╭─────╮             │
│        ╱       ╲            │
│       │   ✦    │            │  ← リングが収縮→バウンス
│        ╲       ╱            │
│         ╰─────╯             │
│                             │
└─────────────────────────────┘
```

**アニメーション**:
- リング: scale 1.0 → 0.9 → 1.1 → 1.0（バウンス）
- 中心: 一瞬明るくフラッシュ
- duration: 200ms

**ハプティクス**（即時）:
- iOS: `.impact(.medium)`
- Android: `VibrationEffect.createOneShot(40, 200)`

### Phase 2: 波紋 + カード出現（200-600ms）

```
┌─────────────────────────────┐
│       ╭─────────────╮       │
│      ╱               ╲      │  ← 波紋1つだけ（シンプル）
│     │    ┌───────┐    │     │
│     │    │  🎵   │    │     │  ← カードがバウンス出現
│      ╲   │ 曲名  │   ╱      │
│       ╰──│      │──╯       │
│          └───────┘          │
└─────────────────────────────┘
```

**アニメーション**:
- 波紋: 1つだけ、scale 0 → 画面端（300ms）
- カード: scale 0.9 → 1.02 → 1.0、opacity 0 → 1（250ms spring）
- 開始: 波紋と同時（200ms時点）

---

## すれ違いカード詳細

### 設計原則：2択に絞る

成立後の次アクションは**「聴く」と「いいね」の2択**に絞る。
詳細画面やコメントは後からでもできるが、曲を聴くことが最も重要な体験。

### レイアウト

```
┌─────────────────────────────────┐
│                                 │
│   ┌────────┐                    │
│   │        │   曲名             │  ← 曲情報を最上位に大きく
│   │   🎵   │   アーティスト名    │
│   │        │                    │
│   └────────┘   👤 ユーザー名さん │  ← 相手情報は控えめ
│                                 │
│   ┌──────────────┐  ┌────────┐ │
│   │   ▶ 聴く     │  │   ♡    │ │  ← 2択：聴く / いいね
│   └──────────────┘  └────────┘ │
│                                 │
└─────────────────────────────────┘
```

### スタイル

```
カードサイズ: 幅90%、高さ auto（コンパクト）
背景: 白（ライト）/ Slate 800（ダーク）
角丸: 20pt
シャドウ: 0 4pt 16pt rgba(0,0,0,0.12)
パディング: 20pt

アルバムアート: 72pt × 72pt、角丸 10pt
曲名: 20pt Bold（最上位階層で大きく）
アーティスト: 15pt Regular、Slate 500
ユーザー名: 13pt Regular、Slate 400
```

### 「聴く」ボタン（プライマリ）

```
┌──────────────┐
│   ▶  聴く    │
└──────────────┘

幅: 親要素の70%
高さ: 48pt
背景: Slate 900（ライト）/ Slate 100（ダーク）
テキスト: 白（ライト）/ Slate 900（ダーク）
```

- **タップ**: Spotify/Apple Music で再生開始
- 未連携時: 連携モーダル表示（ただしカードは消さない）

### 「いいね」ボタン（セカンダリ）

```
┌────────┐
│   ♡    │
└────────┘

幅: 48pt（正方形）
高さ: 48pt
背景: 透明
ボーダー: 1pt Slate 300
```

- **タップ**: いいね追加 + ハートがバウンス + カードが消える
- いいねすると自動的にカードが閉じ、体験完了

### カードを閉じる

- **上スワイプ**: カードを閉じる（履歴には残る）
- **カード外タップ**: カードを閉じる
- **聴く/いいね後**: 1秒後に自動で閉じる

### 階層の明確化

```
階層1（最大）: 曲名         20pt Bold
階層2:       アーティスト    15pt Regular
階層3（最小）: ユーザー名     13pt Regular

※ カード内情報は3階層までに限定
```

# 4. 余韻（Afterglow）

## ホーム画面への反映

すれ違い後、ホーム画面に戻ったときの状態。

```
┌─────────────────────────────┐
│                             │
│  ┌───────────────────────┐  │
│  │  ✨ たった今の出会い   │  │  ← 光るバッジ
│  │                       │  │
│  │  🎵 曲名              │  │
│  │     アーティスト       │  │
│  │              [見る →] │  │
│  └───────────────────────┘  │
│                             │
│         ╭─────╮             │
│        ╱       ╲            │  ← パルスリングに
│       │   ◉    │            │     残光エフェクト
│        ╲   ✨  ╱            │
│         ╰─────╯             │
│                             │
└─────────────────────────────┘
```

### 残光エフェクト

```
すれ違い直後（〜30秒）:
- パルスリングに小さな光の粒子が残る
- 粒子は徐々にフェードアウト
- リングの色がわずかに明るい
```

### 「たった今の出会い」カード

- 通常のリストアイテムではなく、特別なカードとして表示
- 背景にグラデーションまたは微細なグロー
- 30秒〜1分後に通常表示に移行

# 7. サウンドデザイン

## すれ違い成立音

**コンセプト**:
- 2つの音が出会って1つになる
- 柔らかく、心地よい
- 短く印象的（1秒以内）

**構成案**:
```
0ms:    低めのベル音（自分を表す）
200ms:  高めのベル音（相手を表す）
400ms:  2音が重なり合う和音
```

**仕様**:
- 長さ: 800ms
- 音量: システム音量の70%
- マナーモード: サウンドオフ、ハプティクスのみ

# 8. パフォーマンス考慮

## アニメーション最適化

| 要素 | 実装 |
|---|---|
| パルスリング | Lottie / Core Animation |
| 波紋 | CAShapeLayer + CABasicAnimation |
| 粒子エフェクト | SpriteKit（iOS）/ Canvas（Android） |
| カード出現 | UIViewPropertyAnimator / Compose Animation |

## バッテリー配慮

- パルスアニメーションは画面表示時のみ
- バックグラウンドでのアニメーション停止
- 低電力モード時は簡易アニメーションに切り替え

# 9. アクセシビリティ

## VoiceOver/TalkBack

| 状態 | アナウンス |
|---|---|
| 待機中 | 「すれ違いを探しています」 |
| 予兆 | 「近くに誰かがいるかもしれません」 |
| 成立 | 「新しいすれ違いがありました。曲名、アーティスト名」 |

## モーション軽減

「視覚効果を減らす」設定時:
- 波紋アニメーション → シンプルなフェード
- パルスリング → 静的なリング + テキスト変化
- 粒子エフェクト → 無効化
