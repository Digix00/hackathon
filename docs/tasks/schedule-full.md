## 発表向け優先タスク（Phase 4-5 から抜粋）

### 必須（デモに必要）

| タスク | 担当 | 理由 |
|---|---|---|
| ホーム画面 | iOS | デモの起点 |
| すれ違い履歴画面 | iOS | 成果の可視化 |
| 通知機能（FCM/APNs） | Backend + iOS | デモ映え |
| オンボーディング（簡易版） | iOS | 初回起動の印象 |

### あれば良い（時間があれば）

| タスク | 担当 | 優先度 |
|---|---|---|
| 曲再生連携 | iOS | 中 |
| プロフィール編集画面 | iOS | 低 |
| いいね機能 | Backend + iOS | 低 |

### 今回はスキップ

- コメント機能
- 通報・ブロック・ミュート
- タグ機能
- 位置情報すれ違い
- オフライン対応
- ログ解析

## デモシナリオ（想定）

```
1. アプリ起動 → オンボーディング（権限許可）
2. Sign in with Apple でログイン
3. お気に入り曲を設定（Spotify 検索 → 選択）
4. 別の iPhone でも同様にセットアップ
5. 2台を近づける → BLE ですれ違い発生
6. プッシュ通知が届く
7. すれ違い履歴を開く → 相手の曲が表示される
```

**デモ所要時間: 約2-3分**
