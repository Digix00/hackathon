# Issue #001: タイトルやアーティスト名が長い場合の対応

## 概要
楽曲のタイトルやアーティスト名が長い場合、UIレイアウトが崩れたり、テキストが画面外に溢れたりする可能性がある。適切なテキスト表示処理を実装する必要がある。

## 発生する可能性のある問題

### 1. テキストの溢れ
- タイトルやアーティスト名が画面幅を超える場合、レイアウトが崩れる
- 複数行にまたがることで、他のUI要素との重なりが発生する可能性

### 2. 視認性の低下
- 重要な情報が表示されない
- ユーザーが楽曲を識別できない

### 3. デザインの一貫性の欠如
- 長いテキストと短いテキストで見た目が大きく異なる

## 対応方針

### iOS アプリでの実装

#### 1. テキストの省略（Truncation）
```swift
// 1行で表示し、末尾を省略
Text(songTitle)
    .lineLimit(1)
    .truncationMode(.tail)
```

#### 2. スクロール可能なテキスト
```swift
// テキストが長い場合にスクロール可能にする
ScrollView(.horizontal, showsIndicators: false) {
    Text(songTitle)
        .lineLimit(1)
}
```

#### 3. 複数行表示（状況に応じて）
```swift
// 最大2行まで表示し、それ以上は省略
Text(songTitle)
    .lineLimit(2)
    .truncationMode(.tail)
```

#### 4. ツールチップまたはロングプレス時の全文表示
```swift
// タップ時に全文を表示するモーダルやポップオーバーを表示
Text(songTitle)
    .lineLimit(1)
    .truncationMode(.tail)
    .onLongPressGesture {
        // 全文を表示するアラートやシートを表示
    }
```

## 推奨される実装

### プレイヤー画面
- **現在再生中のタイトル**: 最大2行、省略記号（...）で切り捨て
- **アーティスト名**: 1行、省略記号で切り捨て
- **長押しで全文表示**: モーダルまたはツールチップで完全な情報を表示

### リスト表示
- **タイトル**: 1行、省略記号で切り捨て
- **アーティスト名**: 1行、省略記号で切り捨て
- **セルの高さ**: 固定して一貫性を保つ

### カード表示
- **タイトル**: 最大2行、省略記号で切り捨て
- **アーティスト名**: 1行、省略記号で切り捨て

## テスト項目

1. **短いテキスト**: 正常に表示されるか
2. **中程度のテキスト**: 適切に省略されるか
3. **非常に長いテキスト（100文字以上）**: レイアウトが崩れないか
4. **絵文字や特殊文字を含むテキスト**: 正しく処理されるか
5. **日本語、英語、韓国語などの多言語**: すべての言語で適切に表示されるか
6. **改行文字を含むテキスト**: 適切に処理されるか

## 参考資料

### SwiftUI Text Modifiers
- `lineLimit(_:)`: 表示する行数の最大値を設定
- `truncationMode(_:)`: 省略方法を設定（.tail, .head, .middle）
- `minimumScaleFactor(_:)`: テキストを縮小して収める場合の最小スケール
