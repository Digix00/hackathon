# 🎧 すれ違い趣味交換アプリ 概要

## 🎯 目的

**すれ違い趣味交換アプリ**は、近接したユーザー同士が音楽などの趣味情報を交換し、**偶然の出会いや発見を楽しむ**ことを目的としたモバイルアプリです。  
BLE（Bluetooth Low Energy）と位置情報を用いた近接検知を軸に、「すれ違い体験」そのものを価値として提供します。

---

## 🌟 提供価値

- 🔄 **自動マッチング:** 近くにいるユーザーと自動的に趣味情報を交換
- 🔍 **趣味の再発見:** 交換履歴やコメントを通じた、新しい音楽との出会い
- 🎵 **共創体験:** すれ違った人々の歌詞から AI が楽曲を自動生成
- 🛡️ **安全な体験:** プライバシーと安全性に配慮した近接コミュニティの実現

---

## 📱 想定ユーザー体験

1. **設定:** iOS/Androidアプリでログインし、プロフィールや音楽情報を設定
2. **すれ違い:** BLE/位置情報による近接検知で他ユーザーと「すれ違い」成立
3. **歌詞投稿:** すれ違い成立時に、その瞬間を歌詞の一節として投稿（任意）
4. **楽曲生成:** 4〜8人の歌詞が集まると、Google Lyria が自動で楽曲を生成
5. **交流:** 交換された楽曲やプレイリスト、生成された楽曲を閲覧・共有
6. **安心・安全:** 通報・ブロック・ミュート機能による安全なコミュニティ維持

---

## ⚙️ 主な機能

| カテゴリ | 機能詳細 |
| :--- | :--- |
| **アカウント・プロフィール** | ・アカウント管理（Apple/Google 認証）<br>・プロフィール管理（名前、年齢公開範囲、出身地、アイコン） |
| **音楽機能** | ・楽曲・プレイリストの登録<br>・お気に入り登録 |
| **交流機能** | ・すれ違い（BLE/位置情報）による交換履歴<br>・すれ違い後のコメント投稿 |
| **歌詞チェーン（Lyric Chain）** | ・すれ違い成立時に歌詞の一節を投稿<br>・4〜8人の歌詞が集まると楽曲生成トリガー<br>・Gemini で歌詞を分析し、Lyria で楽曲自動生成<br>・生成完了時に参加者全員へプッシュ通知 |
| **安全対策** | ・通報、ブロック、ミュート<br>・Gemini によるコンテンツモデレーション |
| **通知** | ・すれ違い成立通知、楽曲生成完了通知、バッチ通知 |

---

## 🗄️ 主要データ

- 👤 **ユーザー情報**（プロフィール、アバター画像）
- 🎵 **音楽データ**（楽曲、プレイリスト、お気に入り）
- 🤝 **インタラクション**（すれ違い履歴とコメント）
- 🛡️ **システムデータ**（通報・ブロック・ミュートなどの安全対策データ）

---

## 🛠️ 技術構成の概要

- **モバイル:** iOS（Swift） / Android（Kotlin）
- **バックエンド:** Go + Echo
- **データベース:** PostgreSQL（Cloud SQL）
- **認証:** Firebase Auth（Apple/Google ログイン）
- **近接検知:** BLE ＋ 位置情報
- **AI/ML:** Vertex AI（Gemini 1.5 Flash + Lyria）
- **外部連携:** Spotify API / Apple Music API

---

## 🔒 運用・設計方針

> **「プライバシーファーストの設計」と「ユーザー体験の最適化」**

- **匿名性の担保:** 近接 ID は短命化し、追跡リスクを抑制
- **プライバシー保護:** 位置情報はサーバー側でぼかして保存
- **快適なUX:** 通知はバッチ処理でまとめて送信し、連投を防止
- **スコープ（MVP）:** 地図表示、画像共有、再配布機能は初期リリースから除外
