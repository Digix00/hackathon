# GDGoC Japan Hackathon

テーマ・審査基準・提出ルール

## 1. テーマ

「Brand New "Hello World."」

## 2. 審査基準

### a. テーマ性

1. 制作したプロダクトと発表内容に、テーマに合致するという納得感があるか。
2. 制作に至るまでの考え方や方法などに共感できるストーリーがあるか。
3. テーマ「Brand New "Hello World."」に対するアンサーが明確で、「Brand New」なものか。

### b. 制作したプロダクトの再現性

1. プロダクトがデモを行うことができ、目的を達成できるか。
2. プロダクトが多くの人に届き、利用されうる形態・方法を取っているか。
3. エラー処理や負荷対策など、実運用に耐えうる堅牢性・拡張性が考慮されているか。

### c. Google技術をうまく活用できているか

1. Googleの技術を効果的・高度に活用し、優れた機能・体験を実現したか。
2. GoogleのAI技術 (Gemini, Gemma, Vertex AI) を、開発段階やプロダクトの中で有効に活用したか。
3. 総合的に技術的な実現難易度が高く、プロダクトとしての完成度が高いか。

### d. 異端性

- 通常では考えつかない技術スタックを採用しているもの。
- プレゼンテーションが強烈に印象に残るもの。
- 制作方法が奇抜なもの。

## 3. 提出物とルール

審査をスムーズかつ公平に行うため、以下のルールの遵守をお願いします。
以下のルールに従っていない場合、全て無効とみなしますのでご注意ください。
従っていない場合の救済措置は無く、異議申し立ても一切受け付けません。

### 公開用リポジトリ（必須）【提出期限 3/21 12:00】

- Organization内に用意された指定の提出用リポジトリを使用してください。
- 開発自体をOrganization内で行う制約はありませんが、提出は提出用で行ってください。

### 発表用スライド（必須）【提出期限 3/22 9:00】

- 当日の「Presentation Freeze」のタイミングで提出してください。
- スライド内には必ず「技術事項の説明」を含めてください。

### デモ動画（必須）【提出期限 3/21 12:00】

- プロダクトの動作や使い方がわかる動画を提出してください。
- 動画編集は必ずしも必要ではありません。画面録画のみのシンプルなもので構いません。
- 当日のネットワークトラブル等でリアルタイムデモが失敗した場合のバックアップ、および審査員による事後確認のために使用します。

### 著作物利用シート（必須）【提出期限 3/21 12:00】

- 運営側での利用目的や範囲について説明をご確認のうえ、記入・提出をお願いします。

### ドキュメント（任意）【提出期限 3/21 12:00】

- プロダクトのより詳細な仕様などを伝えたい場合は、ReadmeやDocs等を提出してください。
